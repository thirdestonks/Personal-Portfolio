BACKGROUND

The Why

Memory Lane started as a personal passion project — a simple platform where I could store and manage my own images and videos, while practicing S3-compatible storage integration using Backblaze B2.

What began as a backend-focused experiment turned into a full-stack web app built with NestJS and Nuxt 3. The goal was to deepen my knowledge of NestJS while learning how to handle file uploads, access control, and cloud storage at a practical level.

Initially built for my own use, I later saw its potential for others. That’s when I started designing it with customization in mind — dynamic configuration, role-based access, and a scalable structure that could serve more users in the future.
